## Description

Opening Range Breakout strategy. The 9:20вЂ“9:35 window defines the opening range (ORB). After 9:45, the strategy enters long if price breaks above ORB high, or short if it breaks below ORB low. Entries close by 14:45. Stop loss trails using ATR (recommended) so winners run while losers are cut quickly.

Bidirectional (long + short simultaneously possible), up to 3 concurrent positions, scored by relative volatility Г— ATR%, filtered by minimum turnover.

---

## Mathematical Blueprint

**Opening Range:** bars from 09:20 to 09:35 (inclusive)
```
orb_high = max(high) in [09:20, 09:35]
orb_low  = min(low)  in [09:20, 09:35]
atr_orb  = ATR(14) on 1-min bars ending at 09:35
```

**Entry trigger:** first bar after 09:45 where
```
long:  high >= orb_high   (breakout up)
short: low  <= orb_low    (breakout down)
```

Entry no later than 11:30. Time exit at 14:45 (market-order equivalent).

**ATR_TRAIL stop (recommended mode):**
```
initial_sl   = orb_low  - r_val   (long)   |   orb_high + r_val  (short)
r_val        = abs(orb_high - orb_low) * r_factor

trail update each bar:
  long:   peak = max(peak, bar_high)
          sl   = max(sl,  peak - atr_orb * atr_mult)
  short:  peak = min(peak, bar_low)
          sl   = min(sl,  peak + atr_orb * atr_mult)
```

**Candidate scoring (no look-ahead вЂ” rolling shifted by 1 day):**
```
rel_vol = today_volume / rolling_14d_avg_volume
atr_pct = atr_orb / orb_close * 100
score   = rel_vol * atr_pct
```

**Position sizing:**
```
qty      = floor((RISK_PCT * MARGIN_CAP) / r_val)
RISK_PCT = 1.25%  |  MARGIN_CAP = Rs 1,00,000  |  MAX_POS = 3
```

Top `top_k=8` candidates selected daily by score; at most 3 actually fill (position cap).

## Transaction Cost Model (Zerodha MIS)

```
brokerage  = min(trade_value * 0.03%, Rs 20) per leg
STT        = 0.025% of sell-side value only (MIS, delivery rate)
NSE charge = 0.00297% both sides
SEBI levy  = Rs 10 per Cr turnover (both sides)
stamp duty = max(buy_value * 0.003%, Rs 300 per Cr buy)
GST        = 18% on (brokerage + exchange charges + SEBI)
slippage   = 5 bps entry | 3 bps SL exit | 2 bps time exit
```

**Fidelity note:** decision logic is now a faithful replica of rf=0.15. **Universe is dynamic**, not static вЂ” `premarket.py` calls `kite.instruments()` fresh each morning, derives the current F&O-underlying list, and prunes CSVs for symbols no longer in F&O.

### Live-run hardening (2026-07-03)

Two robustness bugs surfaced on the first live sessions (not logic-fidelity issues):

| # | Bug | Symptom | Fix |
|---|---|---|---|
| 12 | `max_positions` not enforced intra-bar | 4 candidates filled on the same 09:45 bar (cap = 3) | Re-check open-count on each candidate inside the entry loop; `break` at the cap |
| 13 | Entry sizing depended on one 09:38 batch LTP fetch, frozen all day | 2026-07-03: baseline's whole batch quote failed в†’ all 8 candidates `no_ltp` в†’ **zero trades that day** | **Size at entry** (`orb_sizing.size_at_entry`) using the actual breakout price, with an `orb_close` floor + immediate per-symbol LTP re-query. Sizing can no longer fail from a data blip; a candidate stays eligible and retries next bar |
| 14 | Position with `token=None` gets **no stop enforcement** | 2026-07-06: OBEROIRLTY entered via size-at-entry, but its Phase-A token lookup had failed (same blip as `no_ltp`) в†’ saved `token=None` в†’ never WS-subscribed в†’ tick-SL blind to it, and the bar fallback was globally disabled while the WS was up в†’ trailing stop never executed; position rode unprotected | (a) **re-resolve token at entry** if Phase A's lookup failed; (b) **per-position bar fallback** вЂ” runs whenever a position isn't tick-eligible (WS down, or `token=None`), gated on eligibility (not live watch-state) to avoid a double-exit race. NB: the WebSocket itself was **healthy** вЂ” 0 WS errors; this was a token/monitoring gap, not a WS failure |

Deployed 2026-07-03 (#12вЂ“13, backup `_bak_20260703_ltp/`) and 2026-07-06 (#14, backup `_bak_20260706_token/`). Sizing **math** unchanged вЂ” only price *source*/*timing* hardened (`_size_one`, price parameterized). Committed to `github.com/anantaam/orb4_n227` @ `ed38558`, `b5771f4`.
